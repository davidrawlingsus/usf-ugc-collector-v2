üìñ How Assets Work

Assets are stored in the database and can be accessed via their UUID. Each asset has a unique URL that serves the file directly from the database.

// Asset URL format: // https://yourdomain.com/assets/{uuid} // Example: // https://yourdomain.com/assets/123e4567-e89b-12d3-a456-426614174000

üñºÔ∏è Using Assets in HTML

You can use assets in your HTML by referencing their UUID in the URL:

<!-- Logo example --> <img src="/assets/logo-uuid-here" alt="Company Logo"> <!-- Background image example --> <div style="background-image: url('/assets/background-uuid-here')"> Content here </div> <!-- Icon example --> <img src="/assets/icon-uuid-here" alt="Icon" style="width: 24px; height: 24px;">

üîß JavaScript Integration

You can dynamically load assets using JavaScript:

// Fetch assets from the API async function loadAssets() { try { const response = await fetch('/api/assets'); const assets = await response.json(); // Find a specific asset by type const logo = assets.find(asset => asset.asset_type === 'logo'); if (logo) { document.getElementById('logo').src = `/assets/${logo.uuid}`; } // Load all images const images = assets.filter(asset => asset.asset_type === 'image'); images.forEach(image => { const img = document.createElement('img'); img.src = `/assets/${image.uuid}`; img.alt = image.name; document.getElementById('gallery').appendChild(img); }); } catch (error) { console.error('Error loading assets:', error); } }

üöÄ Best Practices

  • Use descriptive names: Name your assets clearly (e.g., "Company Logo", "Hero Background")
  • Optimize file sizes: Compress images before uploading for better performance
  • Use appropriate formats: PNG for logos/icons, JPEG for photos, WebP for modern browsers
  • Cache assets: Assets are cached for 1 year, so updates require re-uploading
  • Backup important assets: Keep local copies of critical branding assets
